5. Database Design

Деректер базасын дизайндау - бұл деректерді сақтау, қолдану және бірліктілікті оптималдау жолын қамтитын процес. Бұл үдерістің негізгі мақсаты - кестелерді жасау, олар арасындағы байланыстарды анықтау және база деректерін әрекеттеу мен сұрауларды тиімді өңдеуге мүмкіндік беру.

Нормалдау - бұл деректер базасын дизайндау процесінің маңызды бөлігі. Нормалдау принциптері деректерді қайталанудан қорғауға және база деректерінің құрылымын оңтайландыруға көмектеседі. Нормалдаудың негізгі деңгейлері:

  • 1NF (Бірінші нормалдық форма): Барлық бағандар атомарлы мәндерден тұруы керек.
  • 2NF (Екінші нормалдық форма): Барлық бағандар толық функционалды тәуелді болуы керек.
  • 3NF (Үшінші нормалдық форма): Барлық бағандар тек негізгі кілтке тәуелді болуы керек, басқа бағандарға емес.

Мысал: Студенттер мен курстар туралы деректер базасын дизайндау үшін нормалдау принциптерін қолдану:

CREATE TABLE students (
    student_id INT PRIMARY KEY,
    name VARCHAR(100),
    age INT
);
CREATE TABLE courses (
    course_id INT PRIMARY KEY,
    course_name VARCHAR(100)
);
CREATE TABLE enrollments (
    enrollment_id INT PRIMARY KEY,
    student_id INT,
    course_id INT,
    FOREIGN KEY (student_id) REFERENCES students(student_id),
    FOREIGN KEY (course_id) REFERENCES courses(course_id)
);
